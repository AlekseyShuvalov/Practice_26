# Интеграция, логики и работы с данными

## Назначение
Данный модуль выполняет роль контроллера в архитектуре приложения. Он обеспечивает связь между графическим интерфейсом (gui-prototyp) и вычислительным ядром (backend-prototyp).
Основные задачи модуля:
- Валидация и нормализация входных данных
- Чтение/запись параметров задачи из файлов
- Случайная генерация тестовых полиномов
- Управление циклом алгоритма (пошаговый запуск, переход вперёд/назад, быстрый пропуск)
- Хранение и сериализация истории состояний для визуализации

## Форматы данных и ввод/вывод

```json
{
```

## Логика случайной генерации данных
1. Для режима случайная генерация будет реализован алгоритм создания валидных тестовых данных:
2. Степень полинома: Случайно выбирается от 1 до 8.
3. Коэффициенты: Генерируются случайные вещественные числа в диапазоне [-5.0, 5.0].
4. Интервал: Случайно выбирается в диапазоне [-10.0, 10.0] при условии l<r.
5. Количество ступеней: Случайно выбирается от 2 до 10.

## Управление состоянием и историей
Для реализации требования о возможности вернуться на несколько шагов назад модуль link хранит полную историю эволюции.
Структура данных истории:
```python
class EvolutionHistory:
    def __init__(self):
        self.generations = []  # Список популяций для каждого поколения
        self.best_individuals = []  # Список лучших особей каждого поколения
        self.fitness_history = []  # Значения fitness лучшей особи для графика
        self.current_step = 0  # Текущий индекс для навигации
```
- При каждом шаге алгоритма бэкенд возвращает новую популяцию, и контроллер сохраняет её в self.generations.
- При нажатии кнопок «Назад»/«Вперед» в GUI контроллер не пересчитывает алгоритм, а просто меняет current_step и отдает GUI сохраненный снимок для отрисовки.
