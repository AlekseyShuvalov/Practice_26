# Practice_26
Репозиторий для выполнения учебной практики.

## Постановка задачи
Дана полиномиальная функция $f(x)$ степени <= 8. Необходимо найти параметры ступенчатой функции $g(x)$ с заданным количеством ступеней k на интервале [l, r] для приближения исходной полиномиальной функции.

## Представление данных
1. **Целевая функция $f(x)$**: используется массив вещественных чисел для хранения коэффициентов функции, где индекс элемента соответствует степени x.
2. **Ген**: вещественное число, равное высоте одной ступени.
3. **Генотип**: одномерный массив вещественных чисел, длина которого равна количеству ступеней. Каждый генотип является полноценной ступенчатой функцией.
4. **Популяция**: двумерный массив, состоящий из N генотипов.

## Функция качества
1. **Расчёт ошибки**: $Error = \frac{1}{N} \sum_{i=0}^{N} |f(x_i) - g(x_i)|$
2. **Приспособленность**: $Fitness = \frac{1}{1+Error}$

Для имитации эволюции будут использоваться: турнирный отбор для трёх генотипов, одноточечное скрещивание и мутация с заданной вероятностью.

## Общий алгоритм
1. **Ввод данных**: пользователь программы вводит коэффициенты полиномиальной функции, интервал, количество ступеней и параметры генетического алгоритма (размер популяции, число поколений и шанс мутации). Доступен ручной ввод, случайная генерация данных, чтение из файла.
2. **Генерация начальной популяции**
3. **Цикл эволюции**
	* Оценка приспособленности для каждой особи в популяции.
	* Сохранение лучшей особи.
	* До тех пор, пока не заполнится новая популяция:
		* Турнир для выявления родителей.
		* Скрещивание.
		* Мутация потомков.
		* Добавление потомков в новую популяцию.
	* Замена старой популяции на новую.
4. **Вывод результата**: лучший найденный генотип содержит искомые высоты ступеней.

# Интеграция, логики и работы с данными

## Назначение
Данный модуль выполняет роль контроллера в архитектуре приложения. Он обеспечивает связь между графическим интерфейсом (gui-prototype) и вычислительным ядром (backend-prototype).
Основные задачи модуля:
- Валидация и нормализация входных данных
- Чтение/запись параметров задачи из файлов
- Случайная генерация тестовых полиномов
- Управление циклом алгоритма (пошаговый запуск, переход вперёд/назад, быстрый пропуск)
- Хранение истории состояний для визуализации

## Логика случайной генерации данных
1. Для режима случайная генерация будет реализован алгоритм создания валидных тестовых данных:
2. Степень полинома: Случайно выбирается от 1 до 8.
3. Коэффициенты: Генерируются случайные вещественные числа в диапазоне [-5.0, 5.0].
4. Интервал: Случайно выбирается в диапазоне [-10.0, 10.0] при условии l<r.
5. Количество ступеней: Случайно выбирается от 10 до 35.

## Управление состоянием и историей
Для реализации требования о возможности вернуться на несколько шагов назад модуль link хранит полную историю эволюции.
Структура данных истории:
```python
class EvolutionHistory:
    def __init__(self):
        self.generations = []  # Список популяций для каждого поколения
        self.best_individuals = []  # Список лучших особей каждого поколения
        self.fitness_history = []  # Значения приспособленности лучшей особи для графика
        self.current_step = 0  # Текущий индекс для навигации
```
- При каждом шаге алгоритма бэкенд возвращает новую популяцию, и контроллер сохраняет её в self.generations.
- При нажатии кнопок «Назад»/«Вперед» в GUI контроллер не пересчитывает алгоритм, а просто меняет current_step и отдает GUI сохраненный снимок для отрисовки.
